# Main answer: why same trades keep getting pulled

Your app is not necessarily creating duplicate **Market** rows now. The bigger issue is that the system keeps **refreshing and reprocessing the same active markets** from Polymarket/Kalshi.

Current loop is roughly:

```txt
worker
→ runMarketLoopOnce()
→ runScanner()
→ fetch Polymarket/Kalshi pages
→ upsert existing markets
→ update lastSeenAt / lastSnapshotAt / latestPrice
→ market-loop sorts by lastSeenAt desc
→ same refreshed markets come to top again
→ same candidates can be rescored/requeued after cooldown or stage change
```

So you see the same trades because:

```txt
1. Venue pages are stable.
2. Scanner refreshes existing markets.
3. Refreshed markets look “new” because lastSeenAt updates.
4. Market loop sorts by lastSeenAt desc.
5. The app does not clearly separate:
   - newly discovered markets
   - refreshed markets
   - materially changed markets
   - already processed markets
6. Completed jobs have cooldowns, but candidate-level cooldown/reprocess logic is still incomplete.
```

The fix is not just market dedupe. You need **processing dedupe + reprocess rules + UI filters**.
