# Main answer: why same trades still get pulled

The biggest current reason is **not duplicate Market rows** anymore. You now have `@@unique([venue, externalId])`, so Market-level dedupe is mostly fixed.

The bigger issue is **processing-level repeat**.

Current flow:

```txt
worker
→ market loop
→ scanner
→ Polymarket/Kalshi pages
→ upsert existing markets
→ update lastSeenAt / latestPrice / snapshots
→ candidate gets refreshed
→ market loop sees it again
→ jobs may be requeued depending on stage/cooldown
```

The most serious bug is in:

```txt
src/lib/engine/scanner-upsert.ts
```

Even though the comment says candidate job enqueueing is owned by `market-loop.ts`, this file still updates existing candidates on every market refresh:

```txt
stage: scoreAction === 'SKIP' ? 'SCANNED' : 'TRIAGED'
candidateScore: ...
sourceScanRunId: ...
lastProcessedAt: new Date()
```

That means an already `DECIDED`, `WATCHING`, or even `EXECUTED` candidate can be pushed back into an earlier processing state when the same market is merely refreshed.

So the repeated-market issue is mainly:

```txt
same active markets are fetched again
+ scanner-upsert refreshes candidate state
+ market loop sorts by lastSeenAt
+ stage/cooldown state is weakened
+ candidate gets reprocessed again
```

# Final diagnosis

Your current codebase is now close to a serious prediction-market system, but the biggest remaining bug is:

```txt
scanner-upsert still touches candidate state on every market refresh.
```

That undermines the reprocess gate and is the main reason the same Polymarket/Kalshi markets can keep returning into the workflow.

The second biggest gap is:

```txt
stage jobs are split, but the automatic downstream job chain is incomplete.
```

So markets can be scanned/researched but not reliably progress through:

```txt
research → judge → risk → execute → order tracking
```

The third biggest gap is:

```txt
paper execution is better, but metrics must only count actual filled paper bets.
```

The next target should be:

```txt
Trusted PAPER Mode v1
```

real markets
real orderbook
